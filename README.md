# Peer Feedback Platform

A full-stack web application for anonymous peer feedback in group projects, built for the Shortcut Asia Internship Challenge 2026.

Students submit anonymous reviews for their teammates. Admins manage groups, create feedback forms, register students, and generate AI-powered summaries using Groq (LLaMA 3.3 70B). When a new student is registered, they automatically receive a welcome email with their login credentials.

**Live demo:** https://peer-feedback-platform-fawn.vercel.app
**GitHub:** https://github.com/sahenasalim/peer-feedback-platform

## Features

### Student
- Secure email and password login
- Personal dashboard with stats - forms assigned, completed, summaries received
- View all assigned feedback forms with open/closed status and progress tracking
- Submit anonymous peer feedback per teammate - star rating (1–5), strengths, improvements
- Real-time progress bar tracking completion
- View AI-generated feedback summary with sentiment badge, average rating, strengths, improvements, actionable advice, and a standout quote

### Admin
- Secure admin login with role-based access control
- Create and manage groups
- Register new student accounts - welcome email sent automatically with login credentials 
- Add students to groups
- Create, open, close, and delete feedback forms
- Monitor submission progress per student with live star ratings
- Generate AI summaries per student using Groq (LLaMA 3.3 70B)
- Regenerate or retry failed summaries
- Aggregate stats showing average ratings across all students and forms

### Security
- Passwords hashed with bcrypt (salt rounds: 10)
- Server-side login API — passwords never compared on the client
- Role-based access — admin routes protected, students cannot access admin panel
- Feedback anonymity enforced at the API level — submitter ID stored but never exposed to peers
- Zod validation on every API route
- Exact target matching — feedback must cover every teammate exactly once
- Duplicate submission prevention at both API and database level

### Email
- Welcome email via Resend when admin registers a new student
- Professional HTML email template with login credentials and onboarding steps
- Login button links directly to the live app

## Key feature flows

### Feedback submission

```
Student logs in → opens feedback form
        ↓
Fills rating, strengths, improvements per teammate
        ↓
Frontend validates completeness
        ↓
POST /api/feedback
  → Zod validates shape
  → Form exists and is open?
  → Submitter is a group member?
  → Feedback covers every teammate exactly once?
  → Already submitted before?
        ↓
prisma.$transaction - all entries saved atomically
        ↓
Redirect to dashboard
```

### AI summary generation

```
Admin clicks "Generate summary" for a student
        ↓
POST /api/ai/summarize
  → Fetch all feedback for that student
  → Build structured prompt
  → Send to Groq (LLaMA 3.3 70B) with JSON mode forced
  → Validate every field of AI response
  → Clamp sentimentScore to [-1, 1]
        ↓
prisma.aISummary.upsert — never creates duplicates
        ↓
If Groq fails → save FAILED status → UI shows retry button
Student sees summary on their dashboard
```

### Student registration with email

```
Admin fills name, email, temporary password
        ↓
POST /api/register
  → Zod validates input
  → Check email not already in use
  → Hash password with bcrypt
  → Create account in database
  → Send welcome email via Resend
        ↓
Student can now log in
```

## How to run locally

### Prerequisites
- Node.js 18+
- Groq API key — [console.groq.com](https://console.groq.com)
- Resend API key — [resend.com](https://resend.com)
- PostgreSQL database — [neon.tech](https://neon.tech)

### Steps

```bash
# 1. Clone the repo
git clone https://github.com/sahenasalim/peer-feedback-platform.git
cd peer-feedback-platform

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
# Fill in DATABASE_URL, DIRECT_URL, GROQ_API_KEY, RESEND_API_KEY, NEXT_PUBLIC_APP_URL

# 4. Set up the database
npx prisma migrate dev
npx prisma generate

# 5. Seed demo data
npm run seed

# 6. Run the app
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## Project structure

```
app/
  api/
    auth/login/       → server-side login with bcrypt
    feedback/         → submit anonymous feedback (6-layer validation)
    ai/summarize/     → generate AI summary via Groq
    forms/            → create and manage feedback forms
    groups/           → create groups and add members
    register/         → register new students with welcome email
  admin/              → admin control panel (role-protected)
  dashboard/          → student dashboard
  feedback/[formId]/  → feedback submission form
  login/              → secure login page
components/
  AdminGroupManager   → full admin UI
  AdminGate           → client-side role protection
  FeedbackForm        → per-teammate feedback with draft state
  SummaryCard         → AI summary display with sentiment badges
  StarRating          → star rating component
  ProgressBar         → feedback completion tracker
lib/
  prisma.ts           → singleton Prisma client
  validators.ts       → centralised Zod schemas
  groq.ts             → Groq AI client
  resend.ts           → Resend email client and welcome email template
prisma/
  schema.prisma       → database schema
  seed.ts             → demo data seeder
```
